# 桌面文档问答应用 - 开发总结

## 项目概述

完善桌面文档问答应用，实现完整的RAG流程：文档上传→解析→分块→向量化存储→检索→AI对话，并添加多种检索方式和评估功能。

## 一、遇到的问题与解决方案

### 1. ChromaDB 嵌入接口兼容问题

**问题描述**：ChromaDB 1.5.x 版本中 `embed_query` 返回 `List[List[float]]` 而非 `List[float]`，导致向量检索分数异常。

**解决方案**：
- 在 `EmbeddingFunctionWrapper` 中添加结果展平处理：
```python
embeddings = self.embedding_function(texts)
if embeddings and isinstance(embeddings[0], list) and isinstance(embeddings[0][0], list):
    embeddings = [emb[0] for emb in embeddings]
```

### 2. 中文分词问题

**问题描述**：关键词提取需要使用 bigram 才能正确匹配中文短语。

**解决方案**：
- 在 `BM25Retriever` 和 `QueryExpander` 中使用 `jieba` 的 `jieba.cut(text, bigrams=True)` 进行中文分词
- 同时保留英文的单词边界处理

### 3. 向量检索分数为0

**问题描述**：嵌入模型对中文支持差，导致向量检索效果不佳。

**解决方案**：
- 使用多语言模型 `paraphrase-multilingual-MiniLM-L12-v2` 替代原有模型
- 在 `config/settings.yaml` 中配置：
```yaml
embedding:
  model_name: "paraphrase-multilingual-MiniLM-L12-v2"
```

### 4. ChromaDB collection.get() 返回None

**问题描述**：调用 `collection.get()` 时需要同时指定 `include` 参数，否则返回 `None`。

**解决方案**：
```python
results = collection.get(
    ids=ids,
    include=["documents", "metadatas", "embeddings"]
)
```

## 三、引用溯源功能实现

### CitationInfo 类 (core/ai_chat.py)

```python
class CitationInfo:
    """引用信息"""
    def __init__(self, source: str, score: float, content_preview: str = ""):
        self.source = source      # 文档来源
        self.score = score        # 相关性分数
        self.content_preview = content_preview  # 文本预览
```

### CitationWidget 类 (ui/chat_panel.py)

显示引用来源列表，包含：
- 标题 "📚 引用来源"
- 每个引用的文档来源、相关性分数、文本预览
- 绿色背景样式便于识别

### 功能特点

1. **默认隐藏**：引用默认不显示，节省界面空间
2. **可切换显示**：状态栏有"显示引用/隐藏引用"按钮
3. **跟随状态**：新生成的对话会跟随当前切换状态
4. **全文可选择**：引用内容支持鼠标选择复制

## 四、检索评估指标

| 指标 | 说明 | 用途 |
|------|------|------|
| Hit@K | 前K个结果中包含相关文档的比例 | 快速召回评估 |
| Precision@K | 前K个结果中相关文档的比例 | 精确度评估 |
| Recall@K | 检索到的相关文档占总量的比例 | 召回完整性评估 |
| F1@K | Precision和Recall的调和平均 | 综合评估 |
| MRR | 首个相关文档排名的倒数平均 | 排序质量评估 |
| NDCG | 归一化折扣累计增益 | 多级相关性评估 |

## 五、检索方式对比

支持10+种检索方式：

1. 纯向量检索 (Vector Only)
2. 纯BM25检索 (BM25 Only)
3. 混合搜索 - RRF融合 (Hybrid RRF)
4. 混合搜索 - 加权求和 (Hybrid Weighted)
5. 查询扩展 + 向量 (Query Expansion + Vector)
6. 查询扩展 + BM25 (Query Expansion + BM25)
7. 重排序 - 分数优先 (Rerank Score)
8. 重排序 - 多样性优先 (Rerank Diversity)
9. 查询扩展 + 混合搜索 (Query Expansion + Hybrid)
10. 查询扩展 + 混合搜索 + 重排序 (Query Expansion + Hybrid + Rerank)
11. 查询扩展 + 向量 + 重排序 (Query Expansion + Vector + Rerank)
